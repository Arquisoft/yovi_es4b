[[section-context-and-scope]]
== Context and Scope
This section describes the system boundary: the communication partners (users and external systems),
the domain-level inputs and outputs, and the technical channels used for interaction. It separates
business context (who interacts with the system and what domain information is exchanged) from
technical context (protocols, ports, and runtime deployment aspects).

Motivation: clear context and scope prevent misunderstandings about responsibilities, interfaces,
and deployment requirements.

=== Elements Description
* **Player**: The end-user who interacts with the system through a web browser to play games and view statistics.
* **Bot**: An automated agent that interacts directly with Gamey for gameplay and with Stats for user statistics queries.
* **WebApp**: The front-end application built with React and Vite, serving the user interface and handling user interactions.
* **AuthService**: Dedicated authentication microservice (Node.js + Express) that handles registration, login, stores password hashes in MongoDB and emits/verifies JWTs.
* **StatsService**: Service responsible for player statistics and match history read/query operations.
* **Gamey**: The game engine and bot service implemented in Rust, responsible for game logic and interactions with bots.
* **UserDB**: The database used to persist user password for authentication.
* **Game**: The database used to persist game statistics.

    database "User Authentication DB\n(MongoDB)\nstores password hashes\nport: 27017" as UserDB
    database "Game Stats DB\n(MongoDB)\nport: 27017" as GameDB

    node "Prometheus\nSystem Monitoring\nport: 9090" as Prometheus
    node "Grafana\nDashboards\nhost port: 9091" as Grafana

' --- External interactions ---
Browser --> Gateway : HTTPS (443) / HTTP (80)
Browser --> WebApp : HTTP (dev 5173)

ExternalBot --> Gamey : HTTPS JSON (4000) - game API
ExternalBot --> Stats : HTTPS JSON (3001) - stats API

' --- Internal service communication ---
Gateway --> WebApp : HTTP (internal)
Gateway --> Auth : HTTP JSON (REST) - auth endpoints (JWT)
Gateway --> Stats : HTTP JSON (REST) - user stats and history
Gateway --> Gamey : HTTP JSON (REST)
Auth --> UserDB : TCP (DB driver) - auth data
Gamey --> Stats : HTTP /internal/v1/matches/finished + x-service-token
Stats --> GameDB : TCP (DB driver) - stats data

' --- Monitoring ---
Auth --> Prometheus : HTTP /metrics
Gamey --> Prometheus : HTTP /metrics
Prometheus --> Grafana : HTTP (data source)

note left of UserDB
DB runs in a container with a mounted persistent volume.
end note

note bottom of Gateway
Internal Docker network; only expose the Gateway in production.
end note

@enduml

|===
| Participants        | Input/Output                                            | Channel/Protocol

| Player (Browser)
| Web navigation, gameplay actions, requests to the frontend
| HTTPS (via Reverse Proxy) / HTTP (dev 5173)

| External Bot
| Game actions and statistics queries (separate calls)
| HTTPS JSON (Gamey API and Stats API)

| Reverse Proxy (nginx)
| Incoming HTTP/HTTPS traffic, routing to internal services
| HTTPS externally / HTTP internally

| WebApp (React + Vite + TypeScript)
| User interactions, REST API calls to Auth, Gamey, and Stats
| HTTP (internal REST)

| AuthService (Node.js + Express)
| Login, registration, JWT issuance/verification; stores password hashes in MongoDB
| HTTP (REST) for auth endpoints (`/register`, `/login`, `/verify`) / TCP (MongoDB driver)

| Gamey (Rust Engine + Bot Service)
| Game logic, bot API, and match result reporting to Stats
| HTTP (internal REST)

| StatsService (Node.js + Express)
| User stats/history queries and persistence of finished match outcomes
| HTTP (REST) for `/v1/me`, `/v1/me/history`, `/internal/v1/matches/finished` / TCP (MongoDB driver)

| Databases (Mongo)
| One database for auth identities and another for game statistics/history
| TCP (MongoDB wire protocol)


| Prometheus
| Metrics scraping from Auth and Gamey
| HTTP (/metrics)

| Grafana
| Dashboard visualization, queries to Prometheus
| HTTP (host port 9091 → container port 3000)

|===
